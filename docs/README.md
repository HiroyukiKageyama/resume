### 大阪SES企業（2016/08〜2019/05）
SES企業所属のエンジニアとして、様々な開発現場に派遣される。

#### 交通情報のビッグデータ解析
- 概要
    - 自動車の交通情報データを、顧客の提示した要求に従って解析する業務。
    - 数億~数十億という規模のデータを扱った。
    - この解析が渋滞予測の精度向上につながり、車社会の利便性に寄与できた。
- プロジェクト規模
    - 4人
- 担当した役割
    - 要件定義
        - 顧客の提示資料を細部まで読み込み、不明瞭な点をピックアップして質問し、ブラッシュアップして要件を明確化した。
        - 最初に要求をできる限り明確化したことで、仕様の認識齟齬による手戻りは発生しなかった。
    - インフラ
        - 大量のデータを処理するのに、どのくらいのプロセスを並行で処理する必要があるかを計算し、それに見合ったAWSのリソースを選定した。
    - バックエンド
        - データ整形のロジックを設計・実装。(Groovy)
        - 当初は数人で担当する予定だったが、仕様の理解度が高くミスが起こりにくいということで一人で実装を任された。
- 業務から得た経験・知見
    - 顧客の要望には注意深く考えるとより明確化できる余地が隠れていて、それを明確にすればするほど手戻りのリスクを抑えることができる。
    - 大量のデータ解析には並列処理が不可欠であり、プログラムの設計
    - リソース
